# `@localmaps/contracts`

API contracts for the LocalMaps self-hosted maps platform. This is the
source of truth for the HTTP + WebSocket surface between the Next.js UI
and the Go gateway.

## What lives here

```
contracts/
  openapi.yaml                # <-- THE source of truth. Edit this file.
  ts/
    api.d.ts                  # Generated by openapi-typescript. Committed.
    index.ts                  # Re-exports + narrow schema aliases.
    schemas.ts                # Hand-written zod validators.
    events.ts                 # WebSocket event discriminated union.
    errors.ts                 # ErrorCode const-object + type.
    type-utils.ts             # Equals<A,B>, Expect<T> helpers.
    dist/                     # `pnpm run build` output.
  scripts/
    check-parity.ts           # Enforces schema bijection.
  test/
    schemas.test.ts           # One positive + one negative per schema.
  package.json
  tsconfig.json
  README.md
```

## Contract rules

Every rule below exists to keep the UI, the Go gateway, and the
generated types in lockstep.

1. **OpenAPI is authoritative.** `contracts/openapi.yaml` is the single
   source of truth for paths, schemas, and response shapes. If you need
   a new endpoint or field, add it there first. The primary assistant
   owns this file (see `docs/06-agent-rules.md` R2, R14).

2. **Zod mirrors OpenAPI.** Every response-body schema in the YAML has a
   matching zod validator in `ts/schemas.ts`. Each schema is paired
   with a compile-time `Equals<Inferred, Generated>` assertion; if the
   assertion breaks, fix the zod to match the YAML — never the other
   way around.

3. **Strict by default.** Hand-written object schemas use `.strict()`
   unless the YAML explicitly allows `additionalProperties`. No silent
   `.passthrough()`: unknown keys in a response mean drift, and drift
   must surface.

4. **The generated `api.d.ts` is committed.** Consumers (UI, tests,
   scripts) import types without running `openapi-typescript` at build
   time. Regenerate it any time you edit the YAML.

5. **Changes require updating tests.** If you add or mutate a schema,
   update both its positive and negative fixtures in
   `test/schemas.test.ts`. The parity script enforces that no schema
   goes un-validated.

## How to regenerate / validate

```sh
cd contracts
pnpm install           # first time, or after lockfile changes
pnpm run lint          # redocly lint — must be zero errors
pnpm run typegen       # openapi-typescript openapi.yaml -> ts/api.d.ts
pnpm run test          # vitest (schema fixtures)
pnpm run parity        # checks: every YAML schema is exported twice
pnpm run check         # == lint + typegen + test + parity
pnpm run build         # compiles ts/ to ts/dist/
```

Any of the above failing blocks downstream work — the UI and Go
gateway both assume `ts/api.d.ts` reflects the current YAML.

## Consumer usage

UI:

```ts
import {
  type Region,
  type Job,
  RegionSchema,
  isJobEvent,
  ErrorCode,
  type WsServerEvent,
} from '@localmaps/contracts';

async function fetchRegions(): Promise<Region[]> {
  const r = await fetch('/api/regions');
  const body: unknown = await r.json();
  // Runtime-validate; throws with a useful zod error on drift.
  const parsed = RegionSchema.array().parse((body as { regions: unknown }).regions);
  return parsed;
}
```

## Ownership

- Primary assistant edits `openapi.yaml` and the `docs/` directory.
- Agents (of any phase) edit `ts/` + `scripts/` + `test/` — never the
  YAML — and never outside this package. See
  `docs/06-agent-rules.md`.
